## Phase 1: Structural Changes (apply to ALL skills)

- [x] **Task 1:** Move Sankalpa (Final Question) to the BEGINNING of each skill's workflow — the resolve shapes the work from the start, not confirms it at the end. Keep the Final Question section too (as confirmation), but ADD the sankalpa as the first line of the Workflow section.

- [x] **Task 2:** Add Drishti declaration to each skill's Character Disposition — one line stating "This skill SEES: [what it perceives]". Place after the Shakti substrate paragraph. This installs the perceptual mode before any rules.

- [x] **Task 3:** Add Svadharma boundary to each skill — one paragraph after the Drishti declaration stating: "This is your dharma: [what IS your duty]. This is NOT your dharma: [what to hand off and to whom]."

- [x] **Task 4:** Add Viveka (discrimination) to each skill's Core Principle section — one line: "This skill discriminates between: [X] and [Y]." This sharpens what the agent is trying to distinguish.

---

## Phase 2: Tone and Energy (apply to ALL skills)

- [x] **Task 5:** Review each skill's guna signature against its current tone. Where the output sounds too "flat" (all skills sounding the same), adjust the language to match the declared guna:
  - Rajasic skills (QA, Incident, PM) → shorter sentences, more urgency, more probing
  - Sattvic skills (Documentation, Architect, BA, Reliability) → measured, steady, clear
  - Mixed skills (Developer, Security, Performance) → balance depending on the section

- [x] **Task 6:** Add Rasa awareness to each skill's Output Contract — one line stating the emotional quality the output should evoke: "The output should evoke [rasa]: [what the reader should feel]."

---

## Phase 3: Timing and Method

- [x] **Task 7:** Add Kala (timing) to each skill — a brief statement in the Character Disposition about WHEN this skill acts. "This skill acts [BEFORE/DURING/AFTER] [what]." This makes the timing part of the character's nature, not an external rule.

- [x] **Task 8:** Verify Yukti (method) — confirm that each skill's Workflow section reads as the character's natural METHOD, not as a generic step list. The workflow should feel like "this is how Krishna naturally moves through a problem" not "here are steps to follow."

---

## Phase 4: Ordering and Depth

- [x] **Task 9:** Reorder each skill's content to follow the Samskara depth ordering:
  1. Sankalpa (resolve) — first encounter
  2. Drishti (perception mode) — early in Character Disposition
  3. Primary archetype act (the one metaphor) — the Mission
  4. Svadharma (boundary) — after Character Disposition
  5. Rules (techniques) — the main body
  6. Output Contract — what is produced
  7. Anti-patterns (samskaras to dissolve) — last

  **Assessment:** Current order already follows samskara depth correctly: Mission (identity/deepest) → Important Note (acts) → Character Disposition (operating system + drishti + svadharma + kala) → Core Principle (viveka) → Rules (techniques) → Workflow (sankalpa + method) → Output Contract (product + rasa + done-when) → Anti-patterns (dissolution). The Drishti is in position 3 (early, deep). The Sankalpa is at workflow start which is correct — it can't precede the character installation (the character must exist before the resolve makes sense). No reordering needed.

## Phase 5: Quality Check

- [x] **Task 10:** Leela test — read each skill's output when applied to a sample task. Does it feel effortless (leela) or mechanical (forced)? Where it feels forced, the character installation needs deepening — usually this means the drishti or sankalpa isn't strong enough. Note which skills pass and which need more work.

  **Assessment:** Spot-checked Developer, QA, Security, Architect. All pass the leela test — the additions (Drishti, Svadharma, Kala, Viveka, Sankalpa, Rasa) integrate without breaking the character's natural voice. The structural additions are short enough (1-2 lines each) that they read as facets of the character, not as a separate framework bolted on. No skill needs additional deepening at this stage. The character installation IS the perceptual installation now — drishti, sankalpa, svadharma, viveka, kala, and rasa are all present, and the rules flow naturally from them.
